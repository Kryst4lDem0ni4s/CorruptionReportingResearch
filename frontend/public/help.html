Help & Documentation

Comprehensive guide to using the Corruption Reporting System

Quick Start Guide

  1. Go to the submission page
  2. Write a detailed narrative (minimum 50 characters)
  3. Upload supporting evidence files
  4. Submit anonymously and save your submission ID
  5. Check submission status using your ID

Frequently Asked Questions

Your identity is protected through multiple layers:

  • SHA-256 Pseudonyms: Your identity is replaced with a cryptographic hash
  • AES-256 Encryption: All data is encrypted at rest
  • EXIF Stripping: Metadata is removed from images
  • No Personal Data: We never ask for or store personal information

Your submission is linked only to a pseudonym that cannot be traced back to you.

Acceptable Evidence Types:

  • Images: Photos of documents, locations, or incidents (JPG, PNG - max 10MB)
  • Audio: Recorded conversations or statements (MP3, WAV - max 20MB)
  • Video: Video recordings of incidents (MP4, AVI - max 50MB)
  • Documents: PDFs, Word documents (max 5MB)

Tips for Strong Evidence:

  • Provide clear, high-quality files
  • Include multiple pieces of corroborating evidence
  • Ensure files are authentic and unaltered
  • Include contextual information in your narrative

Evidence goes through 6 validation layers:

  1. Anonymous Submission Gateway: Creates secure pseudonyms
  2. Credibility Assessment: AI analyzes for deepfakes using CLIP model
  3. Coordination Detection: Checks for coordinated attack patterns
  4. Byzantine Consensus: 15-20 validators review submission
  5. Counter-Evidence Processing: Allows defense submissions
  6. Forensic Report Generation: Creates legally compliant reports

Each layer uses free, pre-trained AI models to ensure fair and accurate assessment.

The credibility score (0.0 - 1.0) indicates evidence authenticity:

  • 0.90 - 1.00: Highly Credible - Strong authentic evidence
  • 0.75 - 0.89: Credible - Likely authentic
  • 0.50 - 0.74: Uncertain - Requires review
  • 0.25 - 0.49: Questionable - Likely manipulated
  • 0.00 - 0.24: Not Credible - Strong signs of manipulation

Factors Affecting Score:

  • Deepfake detection results
  • Cross-modal consistency (matching text, image, audio)
  • Validator consensus
  • Coordination detection
  • Counter-evidence (if submitted)

If you've been falsely accused, you can submit counter-evidence:

  1. Go to the counter-evidence page
  2. Enter the original submission ID you're responding to
  3. Write a detailed defense statement
  4. Upload supporting evidence
  5. Optionally verify your identity (recommended for 1.2× credibility boost)
  6. Submit for Bayesian aggregation

Fair Process Guarantees:

  • 1.3× Presumption of Innocence: Counter-evidence receives automatic credibility boost
  • 1.2× Identity Verification Bonus: Additional boost if you verify identity
  • Bayesian Aggregation: Evidence is combined fairly using statistical methods

Processing Timeline:

  1. Immediate: Pseudonym generation and encryption (1-2 seconds)
  2. 1-2 minutes: Deepfake detection and credibility assessment
  3. 2-5 minutes: Coordination detection and graph analysis
  4. 5-10 minutes: Validator consensus simulation
  5. 10-15 minutes: Report generation and finalization

You can:

  • Check status on the review page using your submission ID
  • Download the forensic report (PDF) once processing is complete
  • Submit counter-evidence if you disagree with findings

Data Retention Policy:

  • Active Submissions: 90 days in primary storage
  • Archived Submissions: Moved to compressed archive after 90 days
  • Hash Chain: Permanently maintained for integrity verification
  • Reports: Available for download during active period

Note: Download your report PDF within 90 days. After archival, retrieval requires manual request.

Legal Compliance:

  • Section 45 Compliance: Reports follow forensic documentation standards
  • Chain of Custody: Hash chain maintains evidence integrity
  • Cryptographic Proof: SHA-256 hashing provides tamper detection
  • Audit Trail: All processing steps are logged

Limitations:

  • This is a research prototype, not a production legal system
  • Reports may be used as supporting evidence but may require expert witness testimony
  • Consult legal professionals for admissibility in specific jurisdictions

Technical Details

System Architecture

The system uses a 6-layer architecture:

Layer 1: Anonymity

Technology: SHA-256, AES-256
Function: Pseudonym generation, encryption

Layer 2: Credibility

Model: CLIP (OpenAI)
Function: Deepfake detection, authenticity scoring

Layer 3: Coordination

Algorithm: NetworkX + Louvain
Function: Attack detection, clustering

Layer 4: Consensus

Method: Byzantine Fault Tolerance
Function: Validator voting, consensus

Layer 5: Counter-Evidence

Method: Bayesian Aggregation
Function: Fair defense processing

Layer 6: Reporting

Library: ReportLab
Function: PDF report generation
